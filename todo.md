## Phase 16 ‚Äî Production-Readiness Gaps vs React/Tauri (from platform review)
Tracking the gaps identified in the production-readiness review (see `.kilo/plans/1784171867575-production-readiness-gaps.md`). Ordered by leverage: correctness/churn first, then DX, then ecosystem.

### Critical ‚Äî correctness under churn
- [x] **DOM reconciler with in-place updates** ‚Äî implemented `mount` + `render(view) -> EffectHandle` in `crates/appfront-dom/src/lib.rs`. `reconcile_node`/`reconcile_children` diff the new `UITree` against the mounted `MountedNode` record at node level (reusing `reconcile::reconcile_keys` for keyed `List` children), updating attributes/text/children in place and only unmounting removed subtrees. `mount_router` now reconciles against the previous view via a per-container `MountedNode` side-channel instead of full-replacing on every navigation.
- [x] **`unmount` + resource cleanup (fix listener/memory leak)** ‚Äî `MountedNode` records track every live event `Closure` (in `LIVE_CLOSURES`) and `EffectHandle`. `MountedRoot::unmount` clears listeners (`set_onclick(None)`/`set_oninput(None)`), drops handles, and releases closures for the whole subtree; route reconciliation calls `unmount` on replaced subtrees. Closes the documented open gap in Phase 15.
- [~] **Real-browser / webview end-to-end tests** ‚Äî added `crates/appfront-dom/tests/browser.rs` (`wasm-bindgen-test`: `mount`/`reactive_text`/`unmount`/in-place `reconcile`/subtree-swap) and a `wasm-tests` CI job running them under headless Chromium via `wasm-bindgen-test-runner`. Playwright smoke on `examples/counter-dom`/`counter-webview` still TODO. (Closes the open Phase 15 item partially.)
- [x] **Real component model (props/slots/children/lifecycle)** ‚Äî `#[component]` fns now take a typed props struct as their first argument; `appfront_core::component::Children<Msg>(Vec<UITree<Msg>>)` gives components a children slot they can re-emit into their own tree. `#[component(memo)]` wraps the build in `appfront_core::memoize`, keyed on `PartialEq`-equality of the props argument, returning the cached `UITree` unchanged when props haven't changed (required loosening `memoize`'s bound from `Msg: 'static` to `Msg: Clone + 'static`). Covered in `crates/appfront-core/tests/view_macro.rs` (typed props, children slot, memoized reuse). Lifecycle hooks (mount/unmount callbacks) still not addressed.

### High ‚Äî match React/Tauri UX
- [ ] **Transitions / animation system** ‚Äî no enter/exit transitions (which also need deferred unmount from the item above), no `prefers-reduced-motion` handling; canvas has taffy but nothing for DOM. Add a transition/animation primitive + `<Transition>`-style wrapper.
- [ ] **Forms & controlled inputs** ‚Äî `Input` is one-way (`on_input` ‚Üí `Msg`); no `select`/`checkbox`/`textarea`/`radio` node kinds, no form/validation/`FormData`. Add the missing `NodeKind`s and a `use_form`/controlled-input helper.
- [x] **Webview: real window-state persistence** ‚Äî `manager.rs`'s `CloseRequested` handler now reads each window's live `outer_position()`/`inner_size()`/`scale_factor()` via `wv.window()` and saves that (converted to logical px), instead of re-saving the geometry that was loaded at startup.
- [ ] **Webview: autoupdate integration** ‚Äî `cargo-packager` "generate-updates" is mentioned but `appfront-webview` has no update checker/installer hook (Sparkle/Windows Update/tauri-updater equivalent). Add an update check + apply path.
- [ ] **Webview: menu bar / tray menu builder** ‚Äî `tray.rs` exists; confirm/extend it into a wired menu-bar + tray-menu builder API and connect to the event loop.
- [x] **Server hardening follow-ups** ‚Äî added double-submit-cookie CSRF protection on `POST /command` (`router/csrf.rs`, on by default, only enforced once a client has loaded a document and picked up the cookie ‚Äî `SmartRouterBuilder::csrf`), a `read_rate_limit`-gated Governor layer on the read routes (previously unthrottled), `ETag`/`Cache-Control`/`If-None-Match` caching for the deterministic `crawler_html`/`opengraph`/`ai-schema.json` routes (rendered once via `OnceLock`, `router/caching.rs`) instead of re-rendering every request, and an app-pluggable `SmartRouterBuilder::auth_hook` gate on `POST /command`. `CorsPolicy::Permissive` stays the default (already configurable via `SmartRouterBuilder::cors`, and permissive is intentional/documented for these unauthenticated read routes) ‚Äî a full session/auth story is still open.

### Medium ‚Äî ecosystem
- [ ] **Async/Suspense story** ‚Äî `Resource` requires caller-driven `load_blocking`/`set_result`; add an `async fn` ‚Üí `Resource` bridge, a `<Suspense>` boundary with loading fallback, and cancellation on unmount/reload.
- [ ] **State management at scale** ‚Äî no store pattern (Zustand/Redux-like), no devtools time-travel, no signal persistence (`localStorage`/`IndexedDB`). Add a store helper + persistence binding.
- [ ] **Component library / design system** ‚Äî ready widgets (modal, tabs, dropdown, sortable table, date picker) only partially present (dialog/notify in webview). Build a shared `appfront` component set usable across DOM/canvas/TUI.
- [ ] **Accessibility completeness** ‚Äî add focus management / roving-tabindex for custom components, `aria-live` for `Resource` loading regions, semantic landmark support beyond native tags.
- [ ] **Documentation & examples** ‚Äî only `docs/quickstart.md` + `docs/ai-schema.md`; add an API book, theming guide, "build a real app" tutorial, migration-from-React doc, and a `examples/real-app` (auth + data grid + router + multi-window webview) as the parity demonstration target.
- [ ] **Broader integration test coverage** ‚Äî no `mount ‚Üí click ‚Üí signal update ‚Üí reconcile` integration test, no `view!`+router e2e, no `MCP ‚áÑ DOM` round-trip. Add these so future claims are test-backed.
