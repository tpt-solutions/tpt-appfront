## Phase 16 — Production-Readiness Gaps vs React/Tauri (from platform review)
Tracking the gaps identified in the production-readiness review (see `.kilo/plans/1784171867575-production-readiness-gaps.md`). Ordered by leverage: correctness/churn first, then DX, then ecosystem.

### Critical — correctness under churn
- [ ] **DOM reconciler with in-place updates** — `mount_router` (`crates/appfront-dom/src/lib.rs:64-86`) full-replaces the subtree on every navigation (`while first_child { remove_child }` + `render_node`). Add a `render(view) -> EffectHandle` path that diffs new `UITree` vs mounted tree at node level (reuse `reconcile::KeyedDiff` in `crates/appfront-core/src/reconcile.rs`), updating attributes/children in place; keep keyed `update_list` as the children special case. Route change becomes a single effect re-rendering only changed subtrees.
- [ ] **`unmount` + resource cleanup (fix listener/memory leak)** — every event closure is `.forget()`-leaked (`lib.rs:234,245,625,637`) and `mount`/router `.forget()` the `EffectHandle` (`lib.rs:84`); no `unmount` API exists. Track closures/handles per mounted node and implement `unmount(node)` removing listeners + dropping handles; route reconciliation must call `unmount` on replaced subtrees. (Closes the documented open gap in Phase 15.)
- [ ] **Real-browser / webview end-to-end tests** — zero `wasm-bindgen-test` and zero Playwright; all verification is headless Rust unit tests or `trunk build` smoke. Add `wasm-bindgen-test` browser tests for `mount`/`hydrate`/click/`reactive_text`, plus a Playwright smoke on `examples/counter-dom` and `examples/counter-webview`; wire into CI (needs headless browser / `wasm-bindgen-test-runner`). (Closes the open Phase 15 item.)
- [ ] **Real component model (props/slots/children/lifecycle)** — `#[component]` only auto-fills `class`/`ai.description` + `is_dynamic` and has no props/children/memo; `view!`'s `{ my_component() }` is just `Fn(...) -> UITree`. Add typed props struct, `children: Vec<UITree>` slots, and optional memo keyed on props equality. Biggest DX gap vs React.

### High — match React/Tauri UX
- [ ] **Transitions / animation system** — no enter/exit transitions (which also need deferred unmount from the item above), no `prefers-reduced-motion` handling; canvas has taffy but nothing for DOM. Add a transition/animation primitive + `<Transition>`-style wrapper.
- [ ] **Forms & controlled inputs** — `Input` is one-way (`on_input` → `Msg`); no `select`/`checkbox`/`textarea`/`radio` node kinds, no form/validation/`FormData`. Add the missing `NodeKind`s and a `use_form`/controlled-input helper.
- [ ] **Webview: real window-state persistence** — `manager.rs:387-392` re-saves the *loaded* geometry without reading live `window.outer_position()/inner_size()`; persisted position/size never captures the user's last state. Read live geometry on `CloseRequested` before saving.
- [ ] **Webview: autoupdate integration** — `cargo-packager` "generate-updates" is mentioned but `appfront-webview` has no update checker/installer hook (Sparkle/Windows Update/tauri-updater equivalent). Add an update check + apply path.
- [ ] **Webview: menu bar / tray menu builder** — `tray.rs` exists; confirm/extend it into a wired menu-bar + tray-menu builder API and connect to the event loop.
- [ ] **Server hardening follow-ups** — `CorsLayer` defaults `Permissive`; `POST /command` has no CSRF protection (hydration shell exposes `window.__appfront.post`); no caching layer (SSR every request), no auth/session story, no rate-limit on read routes. Add CSRF token, response caching, auth hooks.

### Medium — ecosystem
- [ ] **Async/Suspense story** — `Resource` requires caller-driven `load_blocking`/`set_result`; add an `async fn` → `Resource` bridge, a `<Suspense>` boundary with loading fallback, and cancellation on unmount/reload.
- [ ] **State management at scale** — no store pattern (Zustand/Redux-like), no devtools time-travel, no signal persistence (`localStorage`/`IndexedDB`). Add a store helper + persistence binding.
- [ ] **Component library / design system** — ready widgets (modal, tabs, dropdown, sortable table, date picker) only partially present (dialog/notify in webview). Build a shared `appfront` component set usable across DOM/canvas/TUI.
- [ ] **Accessibility completeness** — add focus management / roving-tabindex for custom components, `aria-live` for `Resource` loading regions, semantic landmark support beyond native tags.
- [ ] **Documentation & examples** — only `docs/quickstart.md` + `docs/ai-schema.md`; add an API book, theming guide, "build a real app" tutorial, migration-from-React doc, and a `examples/real-app` (auth + data grid + router + multi-window webview) as the parity demonstration target.
- [ ] **Broader integration test coverage** — no `mount → click → signal update → reconcile` integration test, no `view!`+router e2e, no `MCP ⇄ DOM` round-trip. Add these so future claims are test-backed.
